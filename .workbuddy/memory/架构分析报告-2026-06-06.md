### 1.1 分层设计（San I/O 模式）

```
┌─────────────────────────────────────┐
│    应用层 & Labs 实验模块            │
│  (Labs: multiverse, social-feed)    │
├─────────────────────────────────────┤
│  UI 服务层 (matrix-sdk-ui)          │
│  - Timeline (事件聚合视图)           │
│  - RoomListService (房间列表)       │
│  - SyncService (同步协调)           │
├─────────────────────────────────────┤
│  中层客户端 (matrix-sdk)            │
│  - Client (登录/同步/房间管理)      │
│  - Room (房间操作/消息发送)         │
│  - Event 处理                       │
├─────────────────────────────────────┤
│  基础数据层 (matrix-sdk-base) [*]   │
│  - RoomInfo / EventCache             │
│  - StateStore Trait (接口)           │
│  - EventCacheStore Trait             │
├─────────────────────────────────────┤
│  加密层 (matrix-sdk-crypto) [*]     │
│  - E2EE 状态机 (无 I/O)              │
│  - CryptoStore Trait                 │
├─────────────────────────────────────┤
│  存储实现                            │
│  - SQLite Store                      │
│  - IndexedDB Store (Web)             │
│  - Memory Store                      │
└─────────────────────────────────────┘

[*] = San I/O 设计（无网络/存储 I/O，仅定义数据结构和算法）
```

**关键特点**：
- **分离关注点**：基础层专注数据结构，中层处理 API 通信，上层提供高级抽象
- **可测试性**：San I/O 层无副作用，单元测试友好
- **多语言支持**：通过 UniFFI 暴露给 Swift/Kotlin，通过 wasm-bindgen 暴露给 Web

### 1.2 核心组件职责映射

| 组件 | 职责 | 所在层 | 主要方法 |
|------|------|--------|---------|
| **Client** | Matrix 服务器通信入口 | 中层 | `sync()`, `create_room()`, `login()`, `join_room_by_id()` |
| **Room** | 房间状态 & 消息操作 | 中层 | `send()`, `leave()`, `members()`, `timeline()` |
| **Timeline** | 事件时间线高级视图 | UI层 | `items()`, `send_reply()`, 聚合 reactions/edits |
| **RoomListService** | 房间列表管理 | UI层 | 房间过滤、排序、通知 |
| **SyncService** | 同步生命周期管理 | UI层 | 启动/停止/协调 EncryptionSync 和 RoomListSync |
| **StateStore** | 房间状态持久化 Trait | 基础层 | 保存/加载 RoomInfo、成员、权限等 |
| **EventCacheStore** | 事件缓存 Trait | 基础层 | 事件存储、翻页查询 |
| **CryptoStore** | 加密密钥/会话持久化 Trait | 加密层 | 密钥管理、会话恢复 |

### 2.2 核心设计理念：用户主页 = 公开 Room

**关键创新**：不引入新的数据结构，完全复用 Matrix Protocol

```
传统社交产品              Matrix 映射
───────────────          ────────────
用户注册                  ─→ 创建公开 Room（World Readable）
                            Room 名称: "{DisplayName} 的主页"

发布动态                  ─→ room.send(text_message)

关注用户                  ─→ join_room(target_feed_room_id)

取关                      ─→ leave(target_feed_room_id)

信息流                    ─→ 遍历 joined_rooms()

                            按时间排序

评论                      ─→ timeline.send_reply(text, event_id)

转发                      ─→ send_reply(text, original_event_id)
                            到自己的主页
```

```rust
pub async fn post_moment(
    &self,
    text: &str,
    image_urls: &[String],  // 图片链接
) -> Result<String, String> {
    let room = self.get_my_room()?;  // 获取自己的主页 Room
    
    let content = if image_urls.is_empty() {
        RoomMessageEventContent::text_plain(text)
    } else {
        // 构建 Markdown（图片作为链接）
        let md = image_urls.iter()
            .map(|u| format!("![]({})", u))
            .collect::<Vec<_>>()
            .join("\n");
        RoomMessageEventContent::text_markdown(&format!("{}\n\n{}", text, md))
    };
    
    let response = room.send(content).await?;
    Ok(response.event_id.to_string())
}
```

**流程**：
1. 获取主页 Room
2. 构建 MessageEvent
3. 发送到服务器
4. 事件进入 EventCache
5. 所有关注者（成员）可见

#### 信息流聚合

```rust
pub async fn timeline(&self, page_size: u32) -> Result<Vec<Moment>, String> {
    // 1. 收集所有关注 Room + 自己的主页
    let mut room_ids = self.joined_feed_room_ids();
    if let Some(ref id) = self.my_feed_room_id {
        room_ids.push(id.to_string());
    }
    
    // 2. 遍历每个 Room，获取 Timeline
    let mut all: Vec<Moment> = Vec::new();
    for rid in &room_ids {
        let room = self.client.get_room(&rid)?;
        let moments = self.fetch_room_moments(&room, page_size).await?;
        all.extend(moments);
    }
    
    // 3. 全局按时间排序（最新优先）
    all.sort_by(|a, b| b.created_at.cmp(&a.created_at));
    Ok(all)
}

async fn fetch_room_moments(
    &self,
    room: &Room,
    page_size: u32,
) -> Result<Vec<Moment>, String> {
    let timeline = room.timeline().await?;
    let items = timeline.items().await;  // ← 调用 matrix-sdk-ui
    
    let mut moments = Vec::new();
    for item in items.iter().take(page_size as usize) {
        if let Some(event) = item.as_event() {
            // 只提取消息类型事件，过滤 reaction/membership 等
            let text = event.content()
                .as_message()
                .map(|m| m.body().to_owned())
                .ok_or("非消息事件")?;
            
            moments.push(Moment {
                id: event.event_id().to_string(),
                author_id: event.sender().to_string(),
                text,
                created_at: event.timestamp().to_system_time()?,
            });
        }
    }
    
    Ok(moments)
}
```

**特点**：
- O(n) 复杂度（n = 关注数），可通过分页优化
- 依赖 matrix-sdk-ui 的 Timeline 进行事件聚合
- 完全无缓存，每次查询从 StateStore 获取最新状态

#### 关注/取关

```rust
pub async fn follow(
    &mut self,
    user_id: &str,
    feed_room_id: &str,
) -> Result<(), String> {
    // 直接调用 SDK 的 join_room
    let room_id: OwnedRoomId = feed_room_id.try_into()?;
    self.client.join_room_by_id(&room_id).await?;
    Ok(())
}

pub async fn unfollow(
    &mut self,
    feed_room_id: &str,
) -> Result<(), String> {
    let room_id: OwnedRoomId = feed_room_id.try_into()?;
    if let Some(room) = self.client.get_room(&room_id) {
        room.leave().await?;
    }
    Ok(())
}

pub fn get_following(&self) -> Vec<String> {
    // 返回已加入的 feed Room 列表（除自己的主页）
    self.joined_feed_room_ids()
}
```

**关键点**：
- 关注状态 = `joined_rooms()` 列表
- 无需额外数据库，完全由 Matrix SDK 管理
- 支持原生的多设备同步

```rust
        event_id.try_into()?,
    ).await?;
    Ok(())
```

## ✨ 第四部分：高级交互场景

### 4.1 场景 1：多用户信息流聚合

```
初始状态：
  Alice 的主页 Room: !alice:example.com (成员: Alice, Bob, Charlie)
  Bob 的主页 Room: !bob:example.com (成员: Bob, Alice)
  Charlie 的主页 Room: !charlie:example.com (成员: Charlie, Alice)

时刻 T1：Bob 发布动态
  bob.post_moment("今天天气真好！")
    ↓
  Room !bob 新增 Message Event {
      event_id: "$evt_bob_1",
      sender: @bob:hs,
      content: "今天天气真好！"
  }

时刻 T2：Charlie 发布动态
  charlie.post_moment("我同意")
    ↓
  Room !charlie 新增 Message Event {
      event_id: "$evt_charlie_1",
      sender: @charlie:hs,
      content: "我同意"
  }

时刻 T3：Alice 查看信息流
  alice.timeline(20)
    ↓
  遍历 joined_rooms: [!alice, !bob, !charlie]
    ↓
  聚合 moments:
    [
      Moment { author: @charlie, text: "我同意", created_at: T2 },
      Moment { author: @bob, text: "今天天气真好！", created_at: T1 },
      Moment { author: @alice, text: "...", created_at: T0 },
    ]
    ↓
  按 created_at 倒序排列 → 返回
```

**交互特点**：
- O(n) 查询复杂度（n = 关注数）
- 实时一致性由 Matrix 服务器保证
- 支持离线存储（EventCacheStore 缓存）

### 4.4 场景 4：数据一致性

| 失败场景 | 恢复机制 |
|---------|---------|

| 多设备冲突关注 | 关注状态完全由 joined_rooms 驱动，自动一致 |
| 点赞重复 | 相同 (event_id, reaction_key) 自动去重（Matrix 协议） |
| 应用崩溃恢复 | StateStore 持久化，重启自动恢复完整状态 |

## 🔴 第六部分：已知限制 & 改进建议

### 6.1 当前限制（代码中标记）

| 限制 | 影响 | 改进建议 |
|------|------|---------|
| **媒体支持** | Moment 只提取文本，不处理图片/视频解析 | 增强 `fetch_room_moments()` 解析 URL 和标签 |
| **分页实现** | `timeline(page_size)` 按数量截断，不支持偏移 | 实现 `TimelineIterator` 支持 `.next_page()` |
| **昵称显示** | 当前用 user_id，未调用 `get_profile()` 获取 display_name | 缓存 UserProfile，异步更新 |
